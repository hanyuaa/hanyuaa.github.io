Oracle数据库高级查询

sql原则：

  • 书写顺序原则：select from where group by having order by
  • 执行顺序原则：from where group by having select order by

多表查询

  • 1．内连接查询

    特点：查询出多张表中必然有关系的数据

    • 显示内连接

    • 隐式内连接（企业开发中常用的）

    隐式与显示的功能和查询数据的结果都是一样的,语法实现不一样

    使用等值连接/内连接（只能使用=号），显示员工的编号，姓名，部门名，使用表别名简化
    使用非等值连接（不能使用=号，其它符号可以，例如：>=，<=，<>，betwen and等），显示员工的编号，姓名，月薪，工资级别

    内连接查询：只能查询出符合条件的记录
    外连接查询：既能查询出符合条件的记录，也能根据一方强行将另一个方查询出来

  • 2．外连接查询

    特点：某一张表的数据全部查询,令一张表中只查询与之有关系的数据

    • 左外连接(开发中使用较多)

      特点：左表的数据全部查询 右表的数据只查询与左表有关系

    • 右外链接

      ​特点：右表的数据全部查询 左表的数据只查询与右表有关系

分页

rownum:序号 随着查询而产生
rowid:每行数据在磁盘上的物理地址
什么是rownum，有何特点:
1）rownum是oracle专用的关健字
2）rownum与表在一起，表亡它亡,表在它在
3）rownum在默认情况下，从表中是查不出来的
4）只有在select子句中，明确写出rownum才能显示出来
5）rownum是number类型，且唯一连续
6）rownum最小值是1，最大值与你的记录条数相同
7）rownum也能参与关系运算
rownum = 1 有值
rownum < 5 有值
rownum <=5 有值
rownum > 2 无值
rownum >=2 无值
rownum <>2 有值 与 rownum < 2 相同
rownum = 2 无值
8）基于rownum的特性，我们通常rownum只用于<或<=关系运算

集合查询

  • 使用并集运算[union]，查询20号部门或30号部门的员工信息

注意：

  • union：二个集合中，如果都有相同的，取其一
  • union all：二个集合中，如果都有相同的，都取

用where行过滤，查询工资在1000-2000和1500-2500之间的员工信息（方式二）

  • 使用差集运算[minus]，查询工资在1000-2000，但不在1500-2500之间的员工信息（方式一）

集合查询的细节：

  • 集合操作时，必须确保集合列数&集合列类型是对应相同
  • A union B union C = C union B union A
  • 当多个集合操作时，结果的列名由第一个集合列名决定
